Binary Search

用一個心智模型把所有眉角串起來，不要分開背。

1核心觀察

一句話心智模型：[l, r] 是一段「還沒排除、答案可能藏在裡面」的閉區間。 每一步只看正中間那個，就能砍掉一半。所有眉角（<= 的等號、//2、±1）都只是為了讓這個區間定義「自洽」。

前提：陣列已排序。因為排序過，比一次中點就能判斷 target 落在左半還右半，於是每一步都能丟掉一半候選，總共最多 log₂n 步就見底，這就是 O(log n) 的來源。

這一題的變體：找不到時不是回 -1，而是回 l（target 該插進去的位置，也就是 lower bound）。細節在下方第 4 個眉角。

2自己玩一遍

輸入一個 target，按「下一步」看 l（左）、r（右）、m（中）怎麼動。 綠色 = 還沒排除的候選區，灰色 = 已確定不可能、被砍掉的。

目前執行到的程式碼行：

while l <= r:    m = (l + r) // 2    if nums[m] > target:   # 中間太大 → 砍右半        r = m - 1    elif nums[m] < target: # 中間太小 → 砍左半        l = m + 1    else:                  # 命中        return mreturn l                  # 找不到：l 就是插入點

3眉角逐一拆解

為什麼 while l <= r 要有等號？

因為 [l, r] 是閉區間：兩端 l 和 r 指到的元素，都還沒被檢查、都還是候選。

當區間縮到剩最後一個元素時，l == r。這時候還有一個沒看過的元素 nums[l]，它可能正好就是 target。

如果寫成 l < r，當 l == r 時就直接跳出迴圈，漏掉最後一個元素沒檢查，答案可能被吃掉。

判斷法則：更新方式決定區間是開還是閉，區間決定迴圈條件。這題用 l=m+1 / r=m-1（兩端都排除，閉區間 [l, r]），所以條件必須是 l <= r。三者要綁一起，不能混搭。

更新方式區間意義迴圈條件
l=m+1, r=m-1[l, r] 閉l <= r
l=m+1, r=m[l, r) 半開l < r

本題用第一種就好。記一種、記熟、記到自洽，比記三種混在一起更不會錯。

為什麼 m = (l + r) // 2？

取區間正中間。看中間那個，不管結果是大是小，都能一次砍掉一半候選，這就是 O(log n) 的來源（n 個元素最多砍 log₂n 次就見底）。

為什麼用 //（整數除法）？

index 必須是整數。(0 + 3) // 2 = 1，自動無條件捨去。偶數個元素時會取到偏左的中點，沒關係，只要每次都砍掉一塊，遲早收斂。

Python 地雷：// 是整數除法（往下取整），/ 會得到浮點數，不能拿來當 index。JS 要用 Math.floor((l + r) / 2)。

面試加分：l + (r - l) // 2

跟 (l + r) // 2 數學上等價，但避免 l + r 在超大陣列時整數溢位（overflow）。Python 整數無上限不會溢位，但 Java／C++ 會。面試時講得出這點會加分。

重點：//2 不是魔法，就是「砍半」這個動作的實作。每一步丟掉一半 = log 級別的速度。
為什麼是 l = m + 1 / r = m - 1，不是 l = m？

因為 nums[m] 這一格剛剛已經比較過了。既然它不等於 target（等於的話早就 return m 了），就沒理由把它留在候選區，直接從 m+1 或 m-1 繼續。

致命 bug：如果寫成 l = m（不 +1），當區間剩兩格、中點算出來剛好等於 l 時，l 永遠不動，變成無窮迴圈。+1 保證每一步區間一定變小。

這也呼應第一個眉角：「把 m 排除掉」正是讓區間維持閉區間 [l, r] 的原因，所以條件才用 <=。眉角全部連動。

為什麼找不到時 return l（而不是 r、不是 -1）？

每一步都在維持一個不變量（invariant）：

位置已確定的性質
所有 index < l元素都比 target 小
所有 index > r元素都比 target 大

當 l > r、迴圈結束，中間「沒判定」的區間消失，左邊全部 < target、右邊全部 > target。l 剛好卡在分界線上 = 第一個「不小於 target」的位置，這正是 target 該插進去的 index。

普通「找不到回 -1」的 binary search，這題只改最後一行成 return l。因為 l 會自然收斂到 lower bound（第一個 ≥ target 的位置）。

用上面的互動，輸入一個陣列裡沒有的數（例如 7、0、100），跑到結束，看 l 停在哪，它每次都會落在正確的插入點。

怎樣才不算「背誦」？一個自我檢查

能回答這四題，就代表你是推導而非背：

  1. 如果把 r 初始化成 len(nums)（而非 len(nums)-1），其他要跟著改什麼？
    答：區間變半開 [l, r)，條件改 l < r，r = m 不減 1。
  2. l = m 為什麼會無窮迴圈？
    答：中點偏左，l 不動。
  3. 為什麼能保證 O(log n)？
    答：每步砍一半。
  4. 找不到時 l 的物理意義是什麼？
    答：第一個 ≥ target 的位置。

把「條件、初始值、更新、回傳」當成一組會互相牽動的齒輪，而不是四句咒語，這就是脫離背誦的關鍵。

4完整程式碼

class Solution:
    def search(self, nums: list[int], target: int) -> int:
        l, r = 0, len(nums) - 1
        while l <= r:
            m = l + (r - l) // 2   # 防溢位寫法
            if nums[m] > target:
                r = m - 1
            elif nums[m] < target:
                l = m + 1
            else:
                return m
        return l   # 找不到：l 是插入點 / lower bound
最容易踩的雷：迴圈條件的等號（<=）、更新的 ±1、以及回傳的 l 三者是一組。要嘛整組閉區間、要嘛整組半開，不能混搭，否則不是漏檢查就是無窮迴圈。

5複雜度

時間 O(log n)每一步都把候選區間砍一半，n 個元素最多 log₂n 步就縮到空。
空間 O(1)只用了 l、r、m 三個變數，跟輸入大小無關（迴圈版沒有遞迴堆疊）。
