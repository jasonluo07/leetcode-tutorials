Group Anagrams

把「字母組成相同」的字串分到同一組（Medium）

1核心觀察

互為變位詞（anagram）的字串，例如 eat、tea、ate，字元順序不同，但用到的字母和次數完全一樣。

所以只要替每個字串算出一個「字母組成簽名」，同一組會算出同一個簽名，不同組則不同。把簽名當 hash map 的 key，同簽名的字串自然落進同一個籃子，完全不用兩兩比較。

心智模型：把每個字串壓成一個「長度 26 的字母計數」——第 0 格是 a 出現幾次、第 1 格是 b……第 25 格是 z。變位詞的計數一模一樣，就是天然的分組 key。

計數是一個 list，list 不能當 hash map 的 key，轉成 tuple 就可以（tuple 不可變、可雜湊）。

2視覺化過程

輸入：["eat", "tea", "tan", "ate", "nat", "bat"]

逐一處理每個字串（高亮＝正在處理，虛線＝下一個）

這個字串的字母計數簽名（長度 26，只有出現過的字母會被點亮）

hash map（key = 計數簽名，value = 同簽名的字串清單）

3程式碼

def groupAnagrams(strs):
    res = {}                                  # key: 計數 tuple，value: 字串清單
    for s in strs:
        count = [0] * 26                      # a~z 各一格
        for c in s:
            count[ord(c) - ord("a")] += 1      # 累加這個字母的次數
        key = tuple(count)                   # list 不能當 key，轉 tuple
        if key not in res:                 # 籃子不存在就先開一個空 list
            res[key] = []
        res[key].append(s)                    # 丟進對應籃子
    return list(res.values())              # 所有籃子
最容易踩的雷：少了 if key not in res: res[key] = [] 就直接 res[key].append(s)，第一次遇到該 key 時那個籃子還沒被建出來，會噴 KeyError。

4複雜度

時間 O(m · n)m 個字串、平均長度 n。每個字串掃一次做計數，計數陣列固定 26 格（常數），所以總共 O(m · n)。
空間 O(m · n)hash map 要把所有字串存下來當 value。
對照「排序當 key」：用 "".join(sorted(s)) 當 key 更好寫，但每個字串要 O(n log n) 排序，總時間變 O(m · n log n)，比計數法慢。面試手寫計數法、排序法口頭帶過即可。
