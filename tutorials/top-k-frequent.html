Top K Frequent Elements

用「桶排序」找出現次數最高的前 k 個數字，免去排序

1核心觀察

最直覺的做法是數完每個數字的次數後，依次數排序再取前 k 個，但排序要 O(n log n)。這題的「進階」正是要你超越它。

關鍵：一個數字的出現次數最多就是 n（陣列長度），所以次數一定落在 1 到 n 之間，範圍有上限。

範圍有上限，就能拿「次數當索引」：開一排籃子，freq[i] 收集所有「出現 i 次」的數字。放完後籃子本身就依次數排好了，從高次數的籃子往低收 k 個就是答案，全程不用比較排序。

心智模型：這跟 Group Anagrams 的 [0] * 26 是同一招——拿值本身當索引來換取 O(1) 定位。差別只在這裡索引代表「次數」，每格裝的是一串數字。

2視覺化過程

輸入：nums = [1, 1, 1, 2, 2, 3]，k = 2

第 1 幕：數出每個數字出現幾次
計數表（key = 數字，value = 次數）
籃子（索引 = 次數，高次數在上）
答案
已收 0 / 2
點「下一步」開始：第 1 幕先掃一遍 nums，數每個數字的次數。

3程式碼

def topKFrequent(nums, k):
    count = {}                                # 數字 -> 次數
    for n in nums:
        count[n] = count.get(n, 0) + 1

    freq = [[] for _ in range(len(nums) + 1)]  # 索引 0..n，freq[i] 放出現 i 次的數字
    for num, c in count.items():
        freq[c].append(num)                   # 丟進「對應它次數」的籃子

    res = []
    for i in range(len(freq) - 1, 0, -1):     # 從次數最高的籃子往低掃
        for num in freq[i]:
            res.append(num)
            if len(res) == k:               # 湊滿 k 個就回傳
                return res
最容易踩的雷：建籃子要用 [[] for _ in range(n + 1)]，別寫 [[]] * (n + 1)——後者每一格都指向同一個 list，往一格 append 會讓所有格一起變。

4複雜度

時間 O(n)計數、進籃、從籃子收集各掃一遍，全程沒有排序。
空間 O(n)籃子陣列有 n + 1 格，hash map 也跟相異數字個數成正比。

對照「排序」的做法（sorted(count.items(), key=..., reverse=True)）：好寫，但要 O(n log n)，正是這題「進階」要你超越的目標。
