Invert Binary Tree

把整棵二元樹左右翻轉成鏡像（Easy）

1核心觀察

「翻轉」就是把樹照鏡子：每個節點的左右子樹對調，而且要一路換到底。

關鍵在於，每個節點要做的事一模一樣：把自己的左右孩子交換。既然每個節點做同一件事、而且子樹本身也是一棵樹，這就是遞迴的教科書形狀。

心智模型：對每個節點，先換掉自己的左右孩子，再叫左右孩子各自做一樣的事。遞迴的終點（base case）是走到空節點，直接返回。

交換的是「整棵子樹」，不是只換兩個節點的值。換了根的左右孩子，底下所有子孫也跟著被帶到對面。

2視覺化過程

preorder 走訪：先處理目前節點（換左右子樹），再遞迴左、遞迴右。看每個節點被走到時，它的兩棵子樹整個滑到對面。

目前走訪的節點 剛被交換的孩子 完成
按「下一步」開始 preorder 走訪。目前的樹還沒翻轉。
def invertTree(root):    if not root:                          # 空節點：到底了        return None    root.left, root.right = root.right, root.left  # 交換左右孩子    invertTree(root.left)               # 遞迴反轉左子樹    invertTree(root.right)              # 遞迴反轉右子樹    return root

3程式碼

def invertTree(root):
    if not root:
        return None
    root.left, root.right = root.right, root.left  # 一行同時交換
    invertTree(root.left)
    invertTree(root.right)
    return root
最容易踩的雷：交換要同時進行。寫成兩行 root.left = root.right 再 root.right = root.left 會先把左邊蓋掉，兩邊都變成原本的右邊。Python 的 a, b = b, a 是同時賦值，安全；其他語言要用暫存變數。

先交換再遞迴（preorder）或先遞迴再交換（postorder）結果都對，重點是每個節點都要換一次。

4複雜度

時間 O(n)每個節點剛好被走訪並交換一次，n 是節點總數。
空間 O(h)遞迴堆疊深度等於樹高 h。平衡樹 O(log n)，最壞（歪斜成一條鏈）退化成 O(n)。
